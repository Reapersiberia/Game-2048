# Руководство по созданию изображений для Base Mini App

Для публикации в Base app вам понадобятся следующие изображения. Все они должны быть размещены в корне проекта после деплоя.

## Необходимые изображения:

### 1. **icon.png** (Иконка приложения)
- **Размер**: 512x512 пикселей (рекомендуется)
- **Формат**: PNG с прозрачным фоном
- **Назначение**: Иконка мини-приложения в каталоге Base app
- **Совет**: Создайте простую иконку с числом "2048" или стилизованным логотипом игры

### 2. **splash.png** (Экран загрузки)
- **Размер**: 1200x675 пикселей (рекомендуется)
- **Формат**: PNG
- **Назначение**: Показывается при загрузке мини-приложения
- **Совет**: Красивый экран с названием игры "2048" и возможно подзаголовком

### 3. **og.png** (Open Graph изображение)
- **Размер**: 1200x630 пикселей (стандарт OG)
- **Формат**: PNG или JPG
- **Назначение**: Используется для превью при шаринге в соцсетях и Base app
- **Совет**: Привлекательное изображение с названием игры и кратким описанием

### 4. **embed-image.png** (Изображение для embed)
- **Размер**: 1200x630 пикселей (рекомендуется)
- **Формат**: PNG или JPG
- **Назначение**: Показывается в embed-превью поста в Base app
- **Совет**: Может быть таким же, как og.png, или специально оформленным

### 5. **screenshot-1.png** (Скриншот игры)
- **Размер**: 1200x800 пикселей (рекомендуется)
- **Формат**: PNG
- **Назначение**: Скриншот игрового процесса для каталога
- **Совет**: Сделайте скриншот игры в процессе (с плитками на поле)

## Как создать изображения:

### Вариант 1: Онлайн-инструменты
- **Canva**: https://www.canva.com/ (бесплатный план)
- **Figma**: https://www.figma.com/ (бесплатный)
- **Photopea**: https://www.photopea.com/ (онлайн Photoshop)

### Вариант 2: Простые placeholder-изображения
Можно временно использовать простые цветные изображения с текстом "2048" для тестирования, а потом заменить на профессиональные.

### Вариант 3: Скриншот игры
1. Откройте игру в браузере
2. Сделайте скриншот (Win+Shift+S на Windows)
3. Обрежьте и сохраните как PNG

## Размещение изображений:

После создания всех изображений:
1. Поместите их в корень проекта (рядом с `index.html`)
2. Убедитесь, что имена файлов точно совпадают:
   - `icon.png`
   - `splash.png`
   - `og.png`
   - `embed-image.png`
   - `screenshot-1.png`
3. Закоммитьте и запушьте на GitHub
4. Vercel автоматически задеплоит их

## Проверка:

После деплоя проверьте, что все изображения доступны:
- `https://ВАШ-APP.vercel.app/icon.png`
- `https://ВАШ-APP.vercel.app/splash.png`
- `https://ВАШ-APP.vercel.app/og.png`
- `https://ВАШ-APP.vercel.app/embed-image.png`
- `https://ВАШ-APP.vercel.app/screenshot-1.png`

## Быстрый старт (минимальные требования):

Если хотите быстро протестировать, создайте хотя бы:
- **icon.png** (512x512) - обязательно
- **og.png** (1200x630) - для превью

Остальные можно добавить позже.
